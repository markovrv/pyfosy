## 1. Введение

**FOS Importer** — программное средство для автоматизированного импорта вопросов в формате GIFT в информационную систему поддержки обучения (ISS) ВятГУ (`https://iss.vyatsu.ru/kaf/`).

Программа выполняет следующие задачи:

- Авторизацию в системе ISS ВятГУ через автоматизированный браузер Chrome
- Последовательный выбор кафедры, специальности и предмета
- Парсинг GIFT-файла с вопросами (включая коды компетенций и типы контроля)
- Последовательный импорт каждого вопроса с вариантами ответов в систему ФОС

Инструмент предназначен для преподавателей и сотрудников кафедр, которым необходимо загрузить большое количество тестовых вопросов в систему без ручного ввода через веб-интерфейс.

### Ключевые особенности версии 3.0

| Особенность | Описание |
|---|---|
| GUI | Графический интерфейс на базе tkinter, окно 450x750 |
| Автоматизация | Управление браузером Chrome через pyppeteer |
| Асинхронность | Async/await операции в фоновом потоке с собственным event loop |
| Поставка | Единый EXE-файл (PyInstaller, `console=False`) — ничего устанавливать не нужно |
| GIFT | Поддержка формата GIFT с кодами компетенций в заголовках |

## 3. Установка и запуск


1. Скопируйте файл `FOSImporter.exe`.
2. Убедитесь, что Google Chrome установлен на компьютере.
3. Проверьте/отредактируйте файл `.env` (см. раздел [7. Конфигурация и параметры](#7-конфигурация-и-параметры)).
4. Запустите `FOSImporter.exe` двойным щелчком.


## 4. Порядок запуска

### 4.1. Перед первым запуском

1. Убедитесь, что Google Chrome установлен. Путь по умолчанию:
   ```
   C:/Program Files/Google/Chrome/Application/chrome.exe
   ```
2. Если Chrome установлен в другом месте, отредактируйте файл `.env`:
   ```
   CHROME_PATH=D:/MyApps/Chrome/chrome.exe
   ```
3. При необходимости укажите логин в `.env` для автозаполнения:
   ```
   LOGIN=your_login
   ```
   > **Рекомендация:** не сохраняйте пароль в `.env` — вводите его вручную для безопасности.

### 4.2. Запуск и завершение

Дважды щёлкните по `FOSImporter.exe`. Откроется главное окно программы размером 450x750 пикселей, автоматически размещающееся у правого края экрана.

Для завершения работы закройте окно кнопкой **X** в заголовке. Программа автоматически закроет браузер Chrome и освободит ресурсы.

## 6. Основные функции и рабочие процессы

1. Раскройте выпадающий список **Предмет**.
2. Выберите предмет, в который будут импортироваться вопросы.

1. Нажмите кнопку **Обзор...**
2. В диалоге выбора файлов найдите и выберите GIFT-файл (*.gift).

   - Закрывает модальное окно ответов

## 7. Конфигурация и параметры

### 7.1. Файл .env

Конфигурация программы задаётся в файле `.env`, который должен находиться в одной папке с `FOSImporter.exe`.

#### Формат файла

```ini
CHROME_PATH=C:/Program Files/Google/Chrome/Application/chrome.exe
LOGIN=
PASSWORD=
```

#### Параметры

| Параметр | Обязательный | Описание | Значение по умолчанию |
|---|---|---|---|
| `CHROME_PATH` | Нет | Полный путь к исполняемому файлу Google Chrome. Используется прямой слеш `/` как разделитель пути | `C:/Program Files/Google/Chrome/Application/chrome.exe` |
| `LOGIN` | Нет | Логин для автозаполнения поля «Логин» в интерфейсе | Пустая строка |
| `PASSWORD` | Нет | Пароль для автозаполнения поля «Пароль» в интерфейсе | Пустая строка |

#### Примечания

- Программа ищет `.env` рядом с исполняемым файлом (в режиме EXE) или рядом со скриптом `gui_full.py`.
- Если `.env` отсутствует, используются значения по умолчанию.
- Пути к Chrome используют прямой слеш (`/`), а не обратный (`\`), даже на Windows.
- Если Chrome установлен нестандартно (например, Яндекс.Браузер, Chromium), укажите соответствующий путь.


## 8. Работа с данными

### 8.1. Формат GIFT-файла

GIFT (General Import Format Technology) — текстовый формат для представления тестовых вопросов, используемый в LMS-системах (Moodle, ISS и др.).

#### Базовый синтаксис

```
::Коды компетенций::Текст вопроса{
    =Правильный ответ
    ~Неправильный ответ 1
    ~Неправильный ответ 2
    ~Неправильный ответ 3
}
```

#### Элементы формата

| Элемент | Описание |
|---|---|
| `::Заголовок::` | Коды компетенций и типов контроля, разделённые пробелами |
| `Текст вопроса` | Формулировка вопроса (после второго `::` и до `{`) |
| `{` и `}` | Границы блока ответов |
| `=Ответ` | Правильный вариант ответа (может быть несколько) |
| `~Ответ` | Неправильный вариант ответа |
| `// Комментарий` | Строка-комментарий (игнорируется парсером) |

#### Пример файла с вопросами с выбором ответа

```
// Пример вопроса с множественным выбором
::В ОПК-1 ОПК-3::Какой язык программирования вы изучаете?{
    =Python
    ~Java
    ~C++
    ~JavaScript
}
```

#### Пример открытого вопроса (без вариантов ответа)

```
::Т У ПК-1 ПК-2::Сколько будет 2+2?{
}
```

### 8.2. Коды компетенций в заголовках

Заголовок вопроса (между `::` и `::`) содержит коды, которые программа автоматически преобразует в настройки компетенций в ISS.

#### Коды типа контроля

| Код | Расшифровка | Действие в программе |
|---|---|---|
| `В` | Входной контроль | Ставится чекбокс «Входной контроль» |
| `Т` | Текущий контроль | Ставится чекбокс «Текущий контроль» |
| `О` | Остаточные знания | Ставится чекбокс «Остаточные знания» |
| `П` | Промежуточная аттестация | Ставится чекбокс «Промежуточная аттестация» |
| `И` | Итоговая аттестация | Ставится чекбокс «Итоговая аттестация» |

#### Коды типа знаний

| Код | Расшифровка | Действие в программе |
|---|---|---|
| `З` | Знания | Добавляет `#знания` в комментарий вопроса |
| `У` | Умения | Переключает тип вопроса на «Практический», добавляет `#умения` в комментарий |
| `Н` | Навыки | Переключает тип вопроса на «Практический», добавляет `#навыки` в комментарий |

#### Коды компетенций

| Код | Описание |
|---|---|
| `ОПК-N` | Общепрофессиональная компетенция номер N (например, `ОПК-1`, `ОПК-2`) |
| `ПК-N` | Профессиональная компетенция номер N (например, `ПК-1`, `ПК-2`) |

#### Примеры заголовков

| Заголовок | Что импортируется |
|---|---|
| `В ОПК-1 ОПК-3` | Входной контроль + компетенции ОПК-1, ОПК-3 |
| `Т У ПК-1 ПК-2` | Текущий контроль + тип «Практический» (#умения) + компетенции ПК-1, ПК-2 |
| `З ОПК-2` | Знания (#знания) + компетенция ОПК-2 |
| `И Н ОПК-5` | Итоговая аттестация + тип «Практический» (#навыки) + ОПК-5 |

### 8.3. Процесс парсинга GIFT-файла

Парсер (метод `_parse_gift_file`) выполняет следующие действия:

1. Читает файл в кодировке UTF-8.
2. Удаляет пустые строки и строки-комментарии (начинающиеся с `//`).
3. Для каждой строки, начинающейся с `::`, извлекает заголовок и текст вопроса с помощью регулярного выражения:
   ```python
   re.match(r'^::(.*?)::(.*?)\s*{', line)
   ```
4. Строки с `=` или `~` обрабатываются как варианты ответов.
5. Текст варианта очищается от префиксов (знаки `=`, `~`, числовые веса `%...%`).
6. Результат — список словарей:
   ```python
   {
       'type': 'multiple',
       'title': 'В ОПК-1 ОПК-3',      # заголовок (коды компетенций)
       'text': 'Какой язык программирования...',  # текст вопроса
       'options': [
           {'text': 'Python', 'isCorrect': True},
           {'text': 'Java', 'isCorrect': False},
           ...
       ]
   }
   ```

### 8.4. Процесс импорта вопроса

Для каждого вопроса программа выполняет:

1. Нажимает кнопку «Добавить вопрос» на странице ISS
2. Вводит текст вопроса в textarea
3. Обрабатывает коды компетенций из заголовка (режим `auto`):
   - Ставит чекбоксы соответствующих типов контроля
   - При кодах `У` или `Н` переключает вид вопроса на «Практический»
   - Заполняет поле комментария хештегами (`#знания`, `#умения`, `#навыки`)
4. Нажимает кнопку «Сохранить»
5. Если у вопроса есть варианты ответов:
   - Нажимает кнопку «Ответы» (открывает модальное окно)
   - Для каждого варианта: «Добавить» → ввод текста → (отметка «Правильный ответ») → «Сохранить»
   - Нажимает «Закрыть» (закрывает модальное окно)

## 9. Безопасность и лицензирование

### 9.1. Хранение учётных данных

- Логин и пароль могут быть указаны в файле `.env` в открытом виде.
- **Рекомендация**: не храните `.env` в сетевых папках или репозиториях.

### 9.2. Сетевое взаимодействие

Программа работает только с адресом `https://iss.vyatsu.ru/`. Других сетевых запросов не выполняет (за исключением открытия popup-окна генератора `https://n8n.markovrv.ru/` при нажатии кнопки «Сгенерировать файл»).

### 9.3. Безопасность браузера

- Программа запускает Chrome с пользовательским User-Agent.
- Браузер работает в режиме `headless=False` (окно видно пользователю).
- После завершения работы браузер закрывается автоматически.

### 9.4. Обработка ошибок

- Все исключения перехватываются и отображаются в интерфейсе и журнале.
- При ошибке авторизации программа сбрасывает состояние и позволяет повторить попытку.
- При ошибке импорта программа возвращает на основную вкладку ISS.

## 11. Устранение неполадок

### 11.1. Ошибки запуска

| Проблема | Причина | Решение |
|---|---|---|
| EXE не запускается | Антивирус блокирует | Добавьте `FOSImporter.exe` в исключения антивируса |
| EXE не запускается | Отсутствует .env | Создайте `.env` рядом с EXE (см. раздел 7) |
| Окно не появляется | Недостаточно прав | Запустите от имени администратора |

### 11.2. Ошибки авторизации

| Проблема | Причина | Решение |
|---|---|---|
| `Ошибка инициализации браузера` | Chrome не найден по указанному пути | Проверьте `CHROME_PATH` в `.env` |
| `Ошибка авторизации` | Неверный логин/пароль | Проверьте учётные данные в ISS |
| Сайт не открывается | Нет сети / сайт недоступен | Проверьте интернет-соединение, попробуйте открыть `https://iss.vyatsu.ru/kaf/` вручную |
| Авторизация зависает | Медленный интернет | Подождите или перезапустите программу |
| Кафедры не загрузились | Структура сайта изменилась | Обратитесь к разработчику |

### 11.3. Ошибки импорта

| Проблема | Причина | Решение |
|---|---|---|
| `Ошибка чтения GIFT-файла` | Файл не найден или повреждён | Проверьте путь к файлу и его кодировку (UTF-8) |
| Вопрос не импортируется | Некорректный формат GIFT | Проверьте синтаксис: `::Заголовок::Текст{` |
| Ответы не добавляются | Неправильные разделители | Используйте `=` для правильных и `~` для неправильных ответов |
| Импорт прерывается на середине | Сайт вернул ошибку / сессия истекла | Повторите авторизацию и импорт |
| Компетенции не проставляются | Неверные коды в заголовке | Проверьте коды (см. Приложение B) |

### 11.4. Ошибки браузера

| Проблема | Причина | Решение |
|---|---|---|
| Chrome не открывается | Неверный путь в `CHROME_PATH` | Укажите правильный путь. Используйте `/` вместо `\` |
| Страница не загружается | Блокировка антивирусом / брандмауэром | Проверьте настройки безопасности |

## 13. Часто задаваемые вопросы (FAQ)

**В: Нужен ли Python для запуска FOSImporter.exe?**
О: Нет. EXE-файл полностью автономен — все зависимости встроены.

**В: Можно ли использовать программу без интернета?**
О: Нет. Программа требует доступ к `https://iss.vyatsu.ru/` для авторизации и импорта вопросов.

**В: Работает ли программа на macOS или Linux?**
О: Нет. Программа работает только на Windows 10/11 (64-bit).

**В: Сколько вопросов можно импортировать за раз?**
О: Ограничений со стороны программы нет. Ограничение определяется системой ISS и стабильностью соединения. Рекомендуется импортировать партиями по 50-100 вопросов.

**В: Можно ли импортировать вопросы с изображениями?**
О: Нет, текущая версия поддерживает только текстовые вопросы и текстовые варианты ответов.

**В: Что делать, если сессия истекла во время импорта?**
О: Остановите импорт кнопкой «Остановить», перезапустите авторизацию и начните импорт заново. Уже импортированные вопросы будут в системе — проверьте их вручную и удалите дубликаты при необходимости.

**В: Можно ли запустить несколько экземпляров программы одновременно?**
О: Технически возможно, но не рекомендуется, так как каждый экземпляр открывает свой браузер и может конфликтовать при авторизации в ISS.

**В: Как изменить путь к Chrome?**
О: Отредактируйте параметр `CHROME_PATH` в файле `.env`. Используйте прямой слеш `/` как разделитель.

**В: Программа не видит мой Chrome, что делать?**
О: Найдите путь к `chrome.exe` (ПКМ на ярлыке Chrome → Свойства → Объект). Пропишите путь в `CHROME_PATH` в `.env`.

**В: Что означает кнопка «Сгенерировать файл»?**
О: Она открывает веб-форму генератора GIFT-файлов в отдельном окне Chrome. Через эту форму можно создать GIFT-файл, который затем будет загружен через «Обзор...».

## 17. Приложение A: Структура GIFT-файла

### A.1. Общий формат

```
// Комментарий (необязательно)
::Код1 Код2::Текст вопроса{
    =Правильный ответ
    ~Неправильный ответ
}

::Код3::Другой вопрос{
}
```

### A.2. Элементы синтаксиса

| Синтаксис | Описание |
|---|---|
| `// текст` | Однострочный комментарий (пропускается парсером) |
| `:: текст ::` | Заголовок вопроса (коды компетенций) |
| `{` | Начало блока ответов |
| `}` | Конец блока ответов |
| `= ответ` | Правильный вариант ответа |
| `~ ответ` | Неправильный вариант ответа |
| `%вес% ответ` | Ответ с весом (парсер удаляет префикс `%...%`) |

### A.3. Типы вопросов

| Тип | Формат | Пример |
|---|---|---|
| С выбором ответа | Блок `{` содержит `=` и/или `~` | `::З::2+2?{ =4 ~3 ~5 }` |
| Открытый (без вариантов) | Блок `{` пустой | `::У::Напишите код{ }` |

### A.4. Требования к файлу

- Кодировка: **UTF-8** (обязательно)
- Каждый вопрос — отдельный блок `::...::{ ... }`
- Пустые строки между вопросами допускаются (и рекомендуются для читаемости)
- Строки-комментарии начинаются с `//`
- Файлы с расширением `.gift` или `.txt`

### A.5. Пример файла

```
// Вопросы по Python - Блок 1
::З ОПК-2::Какой метод Pandas загружает CSV-файл?{
    =pd.read_csv()
    ~pd.load_csv()
    ~pd.open_csv()
    ~pd.get_csv()
}

::У ОПК-2::Напишите код для удаления строк с NaN{
}

::Т Н ОПК-3 ПК-1::Какой алгоритм используется для кластеризации?{
    =K-means
    ~Random Forest
    ~Линейная регрессия
}
```

## 18. Приложение B: Расшифровка кодов компетенций

### B.1. Коды типа контроля

Используются в заголовке GIFT-вопроса. При импорте программа отмечает соответствующий чекбокс в системе ISS.

| Код | Полное название | Описание |
|---|---|---|
| `В` | Входной контроль | Проверка начального уровня знаний |
| `Т` | Текущий контроль | Проверка в процессе обучения |
| `О` | Остаточные знания | Проверка остаточных знаний |
| `П` | Промежуточная аттестация | Промежуточная проверка |
| `И` | Итоговая аттестация | Финальная проверка |

### B.2. Коды типа знаний/умений/навыков

| Код | Полное название | Действие при импорте |
|---|---|---|
| `З` | Знания | Добавляет `#знания` в комментарий вопроса |
| `У` | Умения | Устанавливает вид вопроса «Практический», добавляет `#умения` в комментарий |
| `Н` | Навыки | Устанавливает вид вопроса «Практический», добавляет `#навыки` в комментарий |

### B.3. Коды компетенций

| Шаблон | Описание | Пример |
|---|---|---|
| `ОПК-N` | Общепрофессиональная компетенция №N | `ОПК-1`, `ОПК-2`, `ОПК-3` |
| `ПК-N` | Профессиональная компетенция №N | `ПК-1`, `ПК-2`, `ПК-3` |
| `ОПК-5` | ОПК номер 5 | `ОПК-5` |

### B.4. Комбинации кодов

Коды в заголовке разделяются пробелами и могут комбинироваться в любом порядке:

```
::Код1 Код2 Код3::Текст вопроса{
    ...
}
```

#### Примеры комбинаций

| Заголовок | Расшифровка |
|---|---|
| `З ОПК-2` | Знания, компетенция ОПК-2 |
| `В ОПК-1 ОПК-3` | Входной контроль, компетенции ОПК-1 и ОПК-3 |
| `Т У ПК-1 ПК-2` | Текущий контроль, умения (практический вопрос), компетенции ПК-1 и ПК-2 |
| `И Н ОПК-5` | Итоговая аттестация, навыки (практический вопрос), компетенция ОПК-5 |
| `П З У ОПК-2` | Промежуточная аттестация, знания, умения, ОПК-2 |
